# koe デザインブリーフ — 見える glass-box コンソール + 音声主役 (2026-06-10)

**この文書が現行の正。** 2026-06-09 の「没入型 orb 一本」(`2026-06-09-immersive-orb-design-brief.md`)は
ユーザー判断で superseded。理由 = Codex App / Hermes Desktop を実機で見て「**情報を orb に隠すより、
サイドバー + ライブ活動パネルで"今なにをしているか"を見せる方が、glass-box 思想にむしろ合致**し、
非エンジニアにも学習可能」と確定 (bd `koe-ios` ピボットノート参照)。

## 外さない原則（ここを守る）
- **音声主役**: 入力の主は声。Codex/Hermes は text 主役だが koe は voice 主役 = text-IDE クローンにしない。
- **活動パネルが主役**: 「今していること(💭思考1行 + 🔧tool + 📎source)」を画面中央〜右に大きく。
  Codex/Hermes は tool 出力を見せるだけ、koe は「検証可能な行為 + ソース」を主役に据える(中心思想)。
- **確信度は既定で非表示**: 常時「たぶん」バッジは出さない(研究 E2 = 生confidence < 作業ログ、user 直感一致)。
  **低確信 × 重大操作(消す/送る/お金)の時だけ**、抽象ラベルでなく**具体的で行動につながる注意**で出す
  (例: 確信度バッジでなく「この送金は取り消せません。確認しますか?」)。生%・常時表示・抽象ヘッジは禁止。
- **OS追従配色**(Windows/Mac のライト・ダーク + アクセント色サンプリング)。両方デザインする。
- **orb は廃止でなく縮小**: 画面の主役からは降りるが、**音声状態インジケータ(小)**として残す。
  2026-06-09 の orb craft(呼吸モーション/二段平滑化/暖色配色)はこのインジケータに流用する(捨てない)。

- **左サイドバー（開閉可）**: 新しい会話 / 検索 / プロジェクト(=文脈・タスクの束。コードベースでない。
  「旅行計画」「確定申告」「毎朝ニュース」等) / メモ・履歴 / オートメーション(定期実行) / 手足tool /
  タスクボード(任意ビュー)。**下部に 残高(時間併記) と ⚙設定**。
- **右メイン**: 上=案内文「今日は何をしましょう?」/ 中=会話 / **下=ライブ活動パネル(主役)** /
  その下に**音声ボタン(=縮小 orb、状態表示)**。

## orb(音声状態インジケータ)の状態表現
小さくなっても状態は一目で(色 + 動きの種類 + 非色の手がかり)。2026-06-09 brief の状態表を縮小要素に適用:
- 待機: ゆっくり脈動・低彩度・「話しかけて」/ 接続中: 収束アニメ・「準備中」+spinner(タイムアウト経過表示) /
  会話中: 音声反応で揺れ・満彩度 / 作業中: 同心円パルス・リング色=リスク段(SAFE中立/CAUTION琥珀/DANGER赤) /
  再接続中: 退色+弱い脈・「再接続中」/ エラー: 赤+アイコン+「再試行」。
- `prefers-reduced-motion` 尊重(明度/彩度の脈のみ)。状態は色だけに頼らずアイコン+文言併記(色覚配慮)。

## ライブ活動パネル（最重要・主役）
- 行: `💭 <1行思考>` / `🔧 <tool名> "<対象>" <✓|spinner|✕>` / `📎 <参照ソース>`。
- 完了は数件だけ薄く残す(全履歴は「履歴」へ)。**orb と同等に作り込む**(高級タイポ・抑制色・glyph+文字)、
  デバッグログにしない。
- **確信度は普段出さない**。低確信×重大時のみ、活動パネル or 承認モーダルに**具体的注意**を一言。

## 承認オーバーレイ（DANGER）
- 危険操作(コマンド/削除/外部送信)の直前、メインの前面に静かなモーダル。
  「◯◯を実行してもいいですか? [許可][拒否]」+ 30s カウントダウン、無操作=拒否(fail-closed)。
- 低確信が絡む時はここに具体的注意を併記(例「送金先は初めてのアカウントです。確認しますか?」)。

## コスト表示（常時・下部・静か）
- 「残高 ¥820 ≈ 約14分」のように**金額 + 目安時間を併記**。
- 課金パッケージ(設計 = bd `koe-3x6`): **月額クレジットプラン**(¥X→¥Yクレジット繰越)を主役 + 自動チャージ補助 +
  BYOK/完全ローカル(無料)を上級。残高低下で自動チャージ、上限到達で停止 → その場で引上げ。

## モデルギャラリー（設定内・カード型 = bd `koe-45n`）
- 各モデルをカードで提示。**評価軸をスペック表示**: 声の自然さ / 反応速度 / tool呼び出しの安定性 /
  作業の安定性 / 機能 / 言語の質。+ 言語ラベル(🇯🇵/🇬🇧/両) + コスト目安 + 提供形態(API/自前/端末内) +
  「非エンジニア向けおすすめ」バッジ + 「おまかせ(自動)」。バー or 3段階で一目比較。

## 視覚グラウンディング（指して話す = bd `koe-jhk`）
- ホットキーで半透明オーバーレイ → 丸/矢印/手書きで囲む + 声「ここ直して」「これ翻訳して」→
  注釈付きスクショを vision 対応モデルへ。既定範囲=前面ウィンドウ。実行は computer_use(DANGER承認で人間ループ)。

## オンボーディング（2ステップ・温かく）
- ① 予算/プラン(月額クレジットプラン or 上限+「明示的に無制限」) ② API キー(BYOK)or「おまかせ(運営)」選択。
- 「端末内に暗号保管・外部送信しない」の安心メッセージ。没入演出より**clear setup**寄り、ただし冷たくしない。

## 設定パネル（統合 = bd epic `koe-0yq`）
- 1パネルに集約: プロバイダ/キー(モデルギャラリー) / 権限(許可フォルダ・URL + per-tool Allow/Ask/Block) /
  手足tool・MCP / 常駐・通知 / 記録・メモリ(閲覧・編集・削除 + on-off) / 音声(声モデル・マイク・barge-in・字幕) /
  表示(OS追従テーマ・ズーム・i18n日英・ペルソナ) / 予算・使用統計 / プライバシー(録音同意・ToS・テレメトリopt-in) /
  診断(koe doctor)。開発者向け(worktree/ファイルブラウザ/YOLO/MDM)は**除外**。

## 維持すべき品質（壊さない）
- コントラスト WCAG AA 以上。Inter 不使用 / purple→blue gradient 不使用。日本語が美しい font-family。
- 暖色配色(`#faf9f5` 系オフホワイト、純白禁止 / 暖色近黒)。低彩度アクセント1色、赤は DANGER/error 専用。
- border-radius・余白・モーション値を均一にしない。input 16px。`prefers-reduced-motion` 尊重。
- 状態・エラー・承認は色だけに頼らずアイコン+文言(色覚配慮)。
